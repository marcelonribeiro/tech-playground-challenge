Getting Started

The API follows RESTful principles. All endpoints return JSON objects. Errors are returned with standard HTTP status codes and a JSON error message.

Base URL
Quick Test (cURL)
BASH curl -X GET http://localhost:5000/api/v1/health
GET /employees

Retrieves a paginated directory of employees. Supports filtering by department and role.

Query Parameters
ParamTypeRequiredDescription
pageintNoPage number (default: 1)
per_pageintNoItems per page (default: 10)
department_idintNoFilter by Department ID
rolestringNoPartial match on role name
Response Example
JSON 
{
  "items": [
    {
      "id": 101,
      "name": "Alice Johnson",
      "email": "alice@pin.com",
      "role": "Data Scientist",
      "department": { "id": 1, "name": "Engineering" },
      "tenure": "Between 2 and 5 years"
    }
  ],
  "total": 50,
  "page": 1,
  "per_page": 10,
  "pages": 5
}
GET /dashboard/company

Returns aggregated KPI metrics for the executive dashboard.

Response Example
JSON 
{
  "total_employees": 150,
  "participation_rate": 85.5,
  "company_enps": {
    "score": 45,
    "classification": "Great",
    "promoters_pct": 55.0,
    "detractors_pct": 10.0,
    "total_responses": 128
  }
}
GET /departments

Helper endpoint to fetch all available departments. Useful for populating dropdowns.

Response Example
JSON 
[
  { "id": 1, "name": "Engineering" },
  { "id": 2, "name": "Human Resources" },
  { "id": 3, "name": "Sales" }
]
GET /analytics/sentiment-overview

Aggregates AI-driven sentiment analysis grouped by feedback category. Used for heatmaps and radar charts.

Query Parameters
ParamTypeRequiredDescription
department_idintNoFilter analysis by department
Response Example
JSON 
{
  "department_id": null,
  "metrics": [
    {
      "field_name": "manager_interaction_comment",
      "friendly_label": "Manager Bond",
      "average_rating": 2.1,
      "sample_size": 45,
      "distribution": {
        "POSITIVE": 5,
        "NEUTRAL": 10,
        "NEGATIVE": 30
      }
    }
  ]
}
